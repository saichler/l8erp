# Plan: Sync Portal Titles with Desktop Section Titles

## Problem

Two types of portal titles need to match the desktop app section title look:

1. **Portal page header** — The top-of-page title bar (e.g., "Employee Self-Service", "Manager Portal"). Currently a thin 56px blue gradient bar with logo, title text, username, and logout button (`<header class="l8-portal-header">`). Should become a full illustrated header matching desktop section titles.

2. **Portal section titles** — The per-section headers shown when navigating within a portal (e.g., "My Profile", "Pay", "Time Off"). Currently a plain `l8-header-frame` with just text and icon, missing SVG illustration, parallax, and subtitle.

Both should match the desktop app section title look: 120px tan gradient, SVG illustration with grid/paths/animated dots, parallax layers, floating icon animation, uppercase title.

## Deduplication

### Section titles
Both portal frameworks currently hardcode header HTML. Instead, they will **call `Layer8SectionGenerator._generateHeader()` directly**, eliminating duplication.

### Page header
The page header generation logic (illustrated header + overlaid username/logout controls) would be identical in both `layer8d-portal.js` and `layer8m-portal.js`. To avoid this duplication, add a **`generatePortalHeader(config)`** method to `Layer8SectionGenerator` in `layer8-section-generator.js`:

```js
// New method in Layer8SectionGenerator
generatePortalHeader: function(config) {
    // config: { title, subtitle, icon, controlsHtml }
    // Returns: illustrated l8-header-frame HTML with controls overlay slot
    return `
    <div class="l8-header-frame parallax-container" style="position:relative;">
        ${this._generateHeader({ title: config.title, subtitle: config.subtitle || '', icon: config.icon || '' }).replace('l8-header-frame parallax-container', 'l8-portal-header-inner')}
        ... SVG bg layer + content layer ...
        <div class="l8-portal-header-controls">${config.controlsHtml || ''}</div>
    </div>`;
}
```

Both portal frameworks then call this single method with their controls HTML:
- `layer8d-portal.js`: passes username span + logout button
- `layer8m-portal.js`: passes menu toggle + refresh button

This keeps all header generation logic in one place (`layer8-section-generator.js`).

## Phase 1: Update Section Generator and Portal Frameworks

### File 1: `l8ui/shared/layer8-section-generator.js`

Add `generatePortalHeader(config)` method that:
- Accepts `{ title, subtitle, icon, controlsHtml }`
- Generates the full illustrated header (reusing `_generateHeader()` internally for the SVG + parallax layers)
- Appends the `controlsHtml` in an absolutely-positioned overlay div (`l8-portal-header-controls`)
- Returns complete HTML

### File 2: `l8ui/portal/layer8d-portal.js`

**Page header**: At init time, call `Layer8SectionGenerator.generatePortalHeader()` with the portal title and username/logout controls HTML. Replace the `l8-portal-header` inner content with the result.

**Section titles**: Replace the inline header HTML in `loadSection()` (lines 103-113) with a call to `Layer8SectionGenerator._generateHeader()`.

### File 3: `l8ui/portal/layer8m-portal.js`

**Page header**: Same — call `generatePortalHeader()` with the mobile controls (menu toggle + refresh button).

**Section titles**: Replace the inline header HTML in `loadSection()` (lines 178-187) with a call to `Layer8SectionGenerator._generateHeader()`.

### File 4: `l8ui/portal/layer8d-portal.css`

Update `.l8-portal-header` to:
- Height: 120px (matching `l8-header-frame`)
- Background: tan gradient (`linear-gradient(135deg, #f5f1ea 0%, #efe8dd 55%, #e4dbcf 100%)`)
- Position: relative (for overlaid controls)
- Overflow: hidden

Add `.l8-portal-header-controls` for the username/logout overlay:
- Position: absolute, right-aligned, z-index above SVG layer
- Dark text to match the section title style

### File 5: `l8ui/portal/layer8m-portal.css`

Similar CSS updates for the mobile page header.
