ERP / Layer 8 Product

One operating system for finance, people, supply chain, projects, manufacturing, and control functions.

ERP by Layer 8 is a multi-domain enterprise platform built as explicit service groups and shared UI primitives, not a stack of unrelated modules behind a common login.

It is designed for operators who need accounting, workforce, inventory, production, analytics, documents, and compliance to behave like one system with shared structure, shared workflows, and shared runtime conventions.

Architecture

Domain services with shared platform conventions and a single operator surface.

ERP is structured as focused service groups for each business domain while sharing Layer 8 runtime services, web infrastructure, reference registries, view factories, popup systems, and navigation rules across the whole product.

Enterprise runtime layout

The point of the architecture is operational coherence. Domain modules stay distinct, but the system is intentionally built so navigation, references, forms, detail views, and shared runtime behavior do not fragment by module.

Modules

The business surface is broad, but the system language stays consistent.

Each module family covers a concrete operational domain. The shared shell and shared component system keep the product readable instead of turning it into twelve disconnected applications.

Financial Management

General ledger, receivables, payables, banking, budgets, assets, tax, cash forecasting, and period-close workflows.

Human Capital

Core HR, payroll, benefits, recruiting, learning, performance, compensation, time, and workforce planning.

Supply Chain

Items, suppliers, purchasing, inventory, warehouses, routes, forecasts, shipments, receiving, and planning services.

Manufacturing

Engineering, BOM, shop floor, work centers, quality, costing, production planning, and execution tracking.

Sales and CRM

Accounts, leads, opportunities, orders, pricing, billing flow, customer service, field activity, and commercial reporting.

Projects and BI

Projects, resources, timesheets, billing, budgets, dashboards, benchmarks, data quality, and reporting models.

Documents and Compliance

Document lifecycles, templates, retention, approvals, risk registers, incidents, audits, and policy controls.

System Operations

Module configuration, import templates, health, logs, admin tooling, and shared operator functions across the platform.

Start

Run the code, inspect the modules, and work from the actual system.

The project is organized as source-first Layer 8 infrastructure. Start locally, inspect the app shell, and then move into the domain services that matter for your use case.

# clone the repository git clone https://github.com/saichler/l8erp.git # move into the project cd l8erp/go/erp/ui/web # serve the web app or wire it into your runtime ./setup-l8ui-submodule.sh
1

Review the app shell

Open `app.html`, shared theme files, and module configs to see how the common desktop surface is assembled.

2

Inspect domain modules

Move into finance, HCM, SCM, manufacturing, projects, BI, and documents to understand service boundaries and UI coverage.

3

Adapt the deployment

Wire the ERP services into your Layer 8 runtime, storage, auth, and deployment model instead of treating the project as a static demo.
